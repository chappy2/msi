Probleme von Javascript

  • beeinflusst von unterschiedlichen Sprachen
    • C (imperativ, {};)
    • Scheme (funktional, )
    • Self (objektorientiert, prototypes)
  • lexikalischer Scope
  • Typerweiterung (Inhärenz) von vielen nicht verstanden

→ CoffeeScript vereinfacht Syntax und verdeckt diese Probleme

Sprachen die CoffeeScript (bzw. Javascript) beinflusst haben: - C: o Weit verbreitete sprache, imperative Porgrammiersparche. Javascript deliberately sieht aus wie C:  {} curly braces. Typischerweise geleehrten sprache c, c++ und java haben soclhe.  ; Semicolon als Line Ending. - Scheme: o General purpose Programmmiersprache (Guy Steele und Gerald Sussman). o Dialekt von LISP (wenig gemeinsam mit C) o Lambda calculus => fällt heutzutage in die Kategorie funktinale programmierung. Inspiriert daher - Self: o Objectorientierte Programmiersprache (David Ungar und Randall Smith). Konzept basiert auf prototypes. War damals anders als bis dahin C++ und Java, basiert auf Klassen. Prototypen flüssiger und dynamischer Weg als Klassen (statischer). o Übernommen:  Kleines mächtiges Set an Operationen/Primitiven tools als design goal. - Auch wenn es eher nach Java oder C++ aussieht, es hat manche Kernideen direkt aus Self und Scheme. Es sieht aus wie eine Sprache, handelt aber anders.
